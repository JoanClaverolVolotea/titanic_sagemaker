# ITER-20260307-01

## Objetivo y contexto
Refactorizar la fase 03 para que la publicacion del pipeline de SageMaker use el bucket del
proyecto y la definicion durable en Terraform, eliminando la dependencia operativa de staging
implicito en el default bucket de SageMaker.

## Decisiones tecnicas y alternativas descartadas
1. La publicacion canonica de fase 03 pasa a ser `pipeline/code/ -> S3 del proyecto -> Terraform`.
2. Se agrego `scripts/publish_pipeline_code.sh` para empaquetar y publicar el bundle versionado
   y los scripts individuales consumidos por la definicion durable del pipeline.
3. `docs/tutorials/03-sagemaker-pipeline.md` se reescribio alrededor de `CodeBundleUri`,
   `terraform plan/apply` y `start_pipeline_execution(...)`.
4. `docs/tutorials/05-cicd-github-actions.md` se ajusto para que el contrato de CI/CD replique
   exactamente el nuevo flujo de fase 03.
5. Alternativa descartada: seguir documentando `pipeline.upsert()` notebook-first como camino
   operativo principal del repositorio.

## IAM usado (roles/policies/permisos clave)
1. No se aplicaron cambios IAM directamente en AWS durante esta iteracion.
2. La documentacion de fase 03 queda alineada con el set del operador:
   `DataScienceObservabilityReadOnly`, `DataSciencePassroleRestricted`,
   `DataSciences3DataAccess` y `DataScienceSageMakerAuthoringRuntime`.
3. El camino canonico documentado ya no requiere permisos adicionales sobre el default bucket de
   SageMaker.

## Comandos ejecutados y resultado esperado
1. Validacion sintactica del helper:
```bash
bash -n scripts/publish_pipeline_code.sh
```
Esperado: sin errores de shell.

2. Validacion rapida de interfaz:
```bash
scripts/publish_pipeline_code.sh --help >/dev/null
```
Esperado: salida de ayuda disponible y codigo de salida 0.

3. Verificacion de whitespace/conflictos en docs y script:
```bash
git diff --check -- docs/tutorials/03-sagemaker-pipeline.md docs/tutorials/05-cicd-github-actions.md docs/tutorials/README.md scripts/publish_pipeline_code.sh
```
Esperado: sin errores.

## Evidencia
1. Nuevo helper local: `scripts/publish_pipeline_code.sh`.
2. Fase 03 reescrita con flujo durable: `docs/tutorials/03-sagemaker-pipeline.md`.
3. Contrato de CI/CD alineado al nuevo publish path:
   `docs/tutorials/05-cicd-github-actions.md`.
4. Roadmap actualizado:
   `docs/tutorials/README.md`.

## Riesgos/pendientes
1. La publicacion real a S3 y el `terraform apply` de `terraform/03_sagemaker_pipeline` no se
   ejecutaron en AWS dentro de esta iteracion.
2. Si cambia `pipeline/code/` y no se rota `CodeBundleUri`, se pierde trazabilidad y puede
   persistir cache inconsistente.
3. Si existe otra fuente de verdad para la definicion del pipeline fuera de Terraform, aparecera
   drift operativo.

   `PipelineExecutionArn`, steps y `ModelPackageArn`.
2. Mantener fase 04 y fase 05 consumiendo exclusivamente el `ModelPackageArn` generado por este
   pipeline durable.
