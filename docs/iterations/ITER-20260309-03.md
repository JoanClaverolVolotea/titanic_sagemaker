# ITER-20260309-03

## Goal

Convertir `docs/tutorials/` en un roadmap autocontenido que use `uv` como unico package
manager Python y que no dependa de rutas del repo fuera del propio folder de tutoriales.

## Change

- Reescritos `docs/tutorials/README.md` y `00-07` para que:
  - usen `uv sync` y `uv run python`
  - creen sus propios archivos bajo `~/titanic-sagemaker-tutorial`
  - no referencien `scripts/`, `config/`, `pipeline/code/`, `vendor/`, `terraform/` ni
    `docs/aws/policies/`
- El roadmap ahora crea sus propios helpers locales:
  - `bootstrap.py`
  - `prepare_splits.py`
  - `prepare_xgb_inputs.py`
  - `evaluate.py`
  - `preprocess.py`
  - `upsert_pipeline.py`
  - `bootstrap_github_actions_role.py`
- La fase 05 ahora incluye un workflow YAML completo de GitHub Actions en lugar de depender
  de archivos externos.

## Why

El lector del tutorial solo recibira `docs/tutorials/`, asi que la documentacion tenia que
dejar de asumir acceso a scripts, manifests o codigo auxiliar del repositorio original.

## Validation

Ejecutado:

1. `rg -n "scripts/|config/|vendor/|terraform/|docs/aws/policies|docs/iterations|pipeline/code/" docs/tutorials`
   sin coincidencias.
2. `rg -n "pip install|uv pip|poetry|conda|virtualenv" docs/tutorials`
   sin coincidencias.
3. `git diff --check`
   sin errores.
