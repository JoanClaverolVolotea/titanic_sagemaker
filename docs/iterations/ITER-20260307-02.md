# ITER-20260307-02

## Objetivo y contexto
Eliminar la dependencia operativa de Terraform del roadmap principal y reemplazarla por un
contrato versionado basado en manifest + scripts idempotentes + SageMaker SDK V3.

## Decisiones tecnicas y alternativas descartadas
1. `config/project-manifest.json` pasa a ser la fuente de verdad local para bucket, roles,
   pipeline, endpoints, imagenes y tags base.
2. `scripts/resolve_project_env.py` reemplaza el uso de `terraform output` en tutoriales y
   scripts auxiliares.
3. `scripts/ensure_project_bootstrap.py` sustituye el bootstrap Terraform para bucket, roles
   SageMaker y Model Package Group.
4. `scripts/upsert_pipeline.py` sustituye la publicacion del pipeline por Terraform y usa
   `pipeline.upsert(...)` con clases V3.
5. `scripts/ensure_github_actions_role.py` cubre el rol del runner via OIDC sin Terraform,
   asumiendo que el proveedor OIDC ya existe en IAM.
6. Alternativa descartada: mantener Terraform solo para fase 03 mientras el resto del roadmap
   se mueve a scripts.

## IAM usado (roles/policies/permisos clave)
1. El operador sigue siendo `data-science-user`.
2. El bootstrap no-Terraform usa `DataScienceS3TutorialBucketBootstrap`,
   `DataSciencePassroleRestricted`, `DataSciences3DataAccess` y
   `DataScienceSageMakerAuthoringRuntime`.
3. El rol de GitHub Actions queda alineado con OIDC y con los patrones `iam:PassRole`
   ya documentados.

## Comandos ejecutados y resultado esperado
1. Resolver manifest:
```bash
python3 scripts/resolve_project_env.py --format json
```
Esperado: contrato consistente de bucket, roles, pipeline y endpoints.

2. Validacion sintactica de scripts Python:
```bash
python3 -m py_compile \
  scripts/resolve_project_env.py \
  scripts/ensure_project_bootstrap.py \
  scripts/ensure_github_actions_role.py \
  scripts/upsert_pipeline.py
```
Esperado: sin errores de sintaxis.

3. Validacion shell del helper de publicacion:
```bash
bash -n scripts/publish_pipeline_code.sh
```
Esperado: sin errores.

4. Verificacion de docs/scripts:
```bash
git diff --check -- config/project-manifest.json scripts docs/tutorials docs/aws/policies/README.md docs/iterations/ITER-20260307-02.md
```
Esperado: sin errores de whitespace o conflictos.

2. `scripts/ensure_github_actions_role.py` requiere que el proveedor OIDC de GitHub exista
   previamente en IAM.
3. No se ejecutaron cambios AWS reales en esta iteracion; solo se implemento el camino local
   y documental.
