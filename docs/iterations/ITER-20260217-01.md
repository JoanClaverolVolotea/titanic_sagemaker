## Objetivo
Agregar politicas IAM base para el usuario operador DS y reorganizar tutoriales en una carpeta dedicada.

## Cambios
- Se creo `docs/aws/policies/` con politicas IAM para operador DS:
  - `01-ds-observability-readonly.json`
  - `02-ds-assume-environment-roles.json`
  - `03-ds-passrole-restricted.json`
  - `04-ds-s3-data-access.json`
  - `05-ds-policy-administration.json`
  - `README.md` con comandos de creacion/asignacion.
- Se movieron los tutoriales de `docs/00..07` hacia `docs/tutorials/00..07`.
- Se creo `docs/tutorials/README.md` con roadmap de fases.
- Se actualizaron referencias internas de "Proximo paso" hacia `docs/tutorials/`.
- Se actualizo `AGENTS.md` para reflejar las nuevas rutas de tutoriales.
- Se estandarizo el esquema de credenciales del operador:
  - Usuario IAM `data-science-user`.
  - Access keys logicas `data-science-user-primary` y `data-science-user-rotation`.
  - Perfiles AWS CLI `data-science-user`, `data-science-user-dev` y `data-science-user-prod`.
- Se actualizo `docs/aws/policies/README.md` con flujo AWS Console para crear keys, aplicar politicas y validar permisos.
- Se actualizaron tutoriales `docs/tutorials/00..07` para usar la misma convencion de credenciales.
- Se agrego mandato explicito: toda operacion AWS del proyecto debe ejecutarse con `data-science-user` como identidad principal (perfiles `data-science-user-dev` y `data-science-user-prod` por entorno).
- Se agrego script `scripts/ensure_ds_policies.sh` para asegurar de forma idempotente que las 4 politicas base existen y estan adjuntas a `data-science-user` (con `DataSciencePolicyAdministration` como prerequisito).

## Evidencia
- Estructura actual de archivos bajo `docs/` valida y consistente.
- Politicas JSON validadas sintacticamente con `python3 -m json.tool`.
- No quedan referencias a rutas antiguas `docs/00..07`.

## Decisiones
- Se mantiene enfoque least-privilege para usuario humano: lectura operativa + `AssumeRole` + `PassRole` restringido.
- Se separa contenido tutorial de historico de iteraciones para mejorar navegacion y mantenimiento.

## Riesgos
- Politicas actualmente fijadas para cuenta `939122281183` y region `eu-west-1`; requieren ajuste si se reutilizan en otro entorno.
- Si no existen los roles objetivo de `AssumeRole`, la politica no sera util hasta crearlos.
