# ITER-20260309-02

## Objetivo y contexto
Reducir `docs/aws/policies/` al set estrictamente necesario para `docs/tutorials/00-07` y
asegurar que cada policy restante pueda adjuntarse y removerse por capacidad.

## Decisiones tecnicas y alternativas descartadas
1. Se eliminan `02-ds-assume-environment-roles.json` y `05-ds-policy-administration.json`
   del directorio de policies del tutorial porque no son requisito para completar las fases.
2. Se mantiene el set minimo en `docs/aws/policies/`:
   `01-ds-tutorial-bootstrap.json`, `02-ds-tutorial-operator.json` y
   `03-ds-tutorial-cleanup.json`.
3. `DataScienceTutorialOperator` deja de incluir `s3:DeleteObject`; la eliminacion de objetos
   queda exclusiva de `DataScienceTutorialCleanup`.
4. El rol de GitHub Actions deja de tener JSON estaticos en `docs/aws/policies/`; el trust y
   la policy inline se generan desde `scripts/ensure_github_actions_role.py`.
5. `scripts/ensure_ds_policies.sh` queda como helper administrativo fuera del flujo tutorial y
   ya no presupone la existencia de una managed policy `DataSciencePolicyAdministration`.
6. Alternativa descartada: mantener `02`, `05`, `11` y `12` en el mismo directorio como
   artefactos opcionales.

## IAM usado (roles/policies/permisos clave)
1. `DataScienceTutorialBootstrap`: bootstrap durable y bootstrap humano de OIDC.
2. `DataScienceTutorialOperator`: fases 01-06 y observabilidad operativa.
3. `DataScienceTutorialCleanup`: stop/delete/reset y limpieza S3.
4. El rol OIDC del runner se converge desde `scripts/ensure_github_actions_role.py`.
