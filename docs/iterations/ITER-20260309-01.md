# ITER-20260309-01

## Objetivo y contexto
Simplificar el set IAM humano del roadmap 00-07 para que el operador trabaje con 3 bundles
principales en lugar del split historico por readonly/passrole/S3/runtime/cleanup/bootstrap.

## Decisiones tecnicas y alternativas descartadas
1. Se colapsan los permisos humanos en `DataScienceTutorialBootstrap`,
   `DataScienceTutorialOperator` y `DataScienceTutorialCleanup`.
2. `DataScienceAssumeEnvironmentRoles` y `DataSciencePolicyAdministration` permanecen como
   opcionales.
3. Se mantienen `11` y `12` para el runner OIDC de GitHub Actions.
4. Se eliminan del repo los JSON legacy del split anterior en lugar de conservarlos como
   artefactos historicos activos.
5. `scripts/check_tutorial_resources_active.sh` se recorta al alcance real del tutorial:
   `s3`, `sagemaker` y `cloudwatch`.
6. `scripts/reset_tutorial_state.sh` pasa a limpiar tambien `pipeline/runtime/` cuando el
   target es `all`.
7. Alternativa descartada: mantener los JSON legacy en paralelo y documentarlos como
   compatibilidad.

## IAM usado (roles/policies/permisos clave)
1. Bootstrap durable: `DataScienceTutorialBootstrap`.
2. Operacion de fases 01-06: `DataScienceTutorialOperator`.
3. Cleanup y reset: `DataScienceTutorialCleanup`.
4. Auto-gestion opcional de policies: `DataSciencePolicyAdministration`.
5. `scripts/ensure_ds_policies.sh` conserva logica para detach de attachments legacy si aun
   existen en AWS.
